# 비기능 수용기준 (NFR)

> 1단계 '구조화' 산출물입니다. `00_sources` 요구사항정의서의 성능요구(PER)·인터페이스요구(SIR)·
> 제약(CONR)을 검증 가능한 비기능 기준으로 정리합니다. 검증 위치까지 함께 못 박습니다.

| 항목 | 목표 | 검증 위치 | 근거 |
| --- | --- | --- | --- |
| 조회 성능 | 시세 통계 조회 P99 300ms 이내 | analytics read model 분리(AC-5) | PER-001 |
| 수집 처리량 | 시군구당 1만 건/분 이상 | ingestion 배치 + numOfRows 1000 페이징 | PER-002 |
| 부하 격리 | 수집 배치 중에도 조회 응답시간 무열화 | CQRS 읽기 분리(AC-5) | PER-003 |
| 회복력 | 외부 API 지연·장애 격리, 부분 수집 허용 | resilience4j 재시도·서킷·타임아웃(AC-2) | SIR-005, SFR-011 |
| 트래픽 한도 | 외부 일일 한도(개발 10,000건) 미초과 | 시군구·계약월 순차/제한 동시 수집으로 호출 조절 | PER-004, CONR-001 |
| 멱등성 | 재수집 중복 0 | 자연키 유니크 제약 + upsert(AC-4) | PER/DAR-003 |
| 가용성 | 핵심 조회 경로 99.9%, 무중단 배포·롤백 | 디스커버리 + 게이트웨이 헬스체크 | PER-005 |

## 제약 전제 (CONR)
- 외부 응답은 XML이며 모든 값이 문자열로 전달된다. 숫자·날짜 형 변환 책임은 시스템에 있다. (CONR-002)
- 거래금액은 만원·콤마·선행 공백 문자열이다. 원 단위 변환은 시스템 책임이다. (CONR-003 → AC-3)
- 동(棟)·등기일자는 등기 완료 건만 채워지고, 해제는 사후 표기될 수 있어 결측·전환을 허용한다. (CONR-004·005)
- 스택은 Java 21 / Spring Boot 3.5.x / Spring Cloud 2025.0 고정, 회복력은 resilience4j로 한정한다. (CONR-006·007)

> 인증키는 코드·이미지·Config 파일에 평문으로 두지 않습니다. 환경변수로만 주입합니다(상세는 `09_security`).
