# 보안 계획 (Security Planning)

> 1단계 '구조화' 산출물입니다. `00_sources` 요구사항정의서의 보안요구(SECR)·제약(CONR)과
> API 공개명세의 인증·라이선스 항목을 외부 공개 API 연계 보안 포스처로 정리합니다.

## 인증키 관리 (SECR-001·002·003·005)
- data.go.kr `serviceKey`는 개인 발급 자격증명입니다. 코드·이미지·Config 파일·git에 평문 저장을 금지합니다. (SECR-001)
- 주입 경로는 환경변수 `MOLIT_SERVICE_KEY` 하나로 통일합니다. Config 서버는 `${MOLIT_SERVICE_KEY:}`로 참조만 합니다. (SECR-002)
- 로그·트레이스·에러 응답에 인증키가 출력되지 않도록 마스킹합니다. (SECR-003)
- 인증키 만료·교체는 설정 갱신으로 무중단 반영합니다. (SECR-005)

## 데이터 보호 (SECR-004)
- 부동산 실거래가는 공개 데이터이며 거래 당사자 식별정보를 포함하지 않습니다. 동(棟)·매도자/매수자 구분은 비식별 범주값입니다.
- 수집·적재 전 구간에서 추가 개인정보가 섞이지 않도록 표준 스키마(AptTransaction)만 적재합니다.
- 출처(국토교통부·공공데이터포털) 표기를 유지합니다(라이선스: 무료·이용허락범위 제한 없음).

## 감사·추적
- 수집 트리거(시군구·계약월·요청자)와 적재 건수를 감사 로그로 남깁니다.
- 외부 호출 실패·서킷 개방·트래픽 초과(resultCode 22) 이벤트를 운영 지표로 노출합니다.
